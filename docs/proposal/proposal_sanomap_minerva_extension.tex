\section{A statement of the research problem}
Biomedical data is inherently multimodal. It spans everything from raw cell sequences to clinical imaging. Recent advances in natural language processing (NLP) and methodologies for artificial intelligence (primarily the emergence of transformers and subsequent generative models) have enabled the creation of unified biomedical knowledge graph bases by extracting large amounts of data from verified scientific literature. The SanoMap project, specifically through the MINERVA (Microbiomes Network for Research and Visualization Atlas) platform, is a prominent example of this, successfully mapping associations between gut microbes and systemic diseases using sophisticated text-mining.

Building upon this foundational work, the delivered system addresses a critical structural gap in MINERVA's graph: it maps the endpoints of microbes to diseases but does not incorporate the intermediary imaging phenotypes that quantify structural tissue changes. Integrating these quantitative imaging phenotypes bridges this gap. Radiomics extracts compact quantitative data from medical scans, providing vital biomarkers for oncology and systemic metabolic conditions such as sarcopenia and nonalcoholic fatty liver disease. Clinical studies increasingly demonstrate that microbiome activity influences these specific tissue-level changes, making radiomic and body-composition features natural intermediary nodes in the knowledge graph.

Standard text-mining pipelines capture findings that authors highlight in prose, but comprehensive ground-truth data is frequently represented in visual formats. Correlation heatmaps, for instance, contain matrices linking dozens of microbial taxa to radiomic feature values—data that NLP alone is not structurally designed to parse. The delivered system addresses both tracks: text-based extraction of phenotype-to-disease associations, and a figure-aware track that extracts quantitative r-values from gradient-colormap heatmaps using a vision language model paired with deterministic pixel-level verification. The two central research questions this system answers are: (1) Can a hybrid AI pipeline (LLMs + VLMs with deterministic verification) reliably extract quantitative correlation coefficients directly from complex medical figures? and (2) Does adding radiomics as an ontological layer increase the clinical completeness of the Neo4j knowledge graph?

\section{Delivered System}

\subsection{Graph Schema}

The delivered system extends MINERVA with a radiomics-first imaging phenotype layer. Following Professor Do's guidance to ``connect disease — body location — imaging modality — representative images,'' the graph implements the following node types:

\begin{itemize}
    \item \texttt{Microbe} and \texttt{MicrobialSignature} — microbial entity nodes from literature NER
    \item \texttt{RadiomicFeature} — texture, morphological, and intensity imaging features (GLCM, wavelet, first-order)
    \item \texttt{BodyCompositionFeature} — quantitative tissue phenotypes extracted from imaging (sarcopenia, visceral adipose tissue, skeletal muscle index, BMI, liver fat fraction)
    \item \texttt{Disease} — disease entity nodes from literature NER and text co-mention extraction
    \item \texttt{BodyLocation} — anatomical sites associated with feature measurements (liver, colon, brain, kidney, heart, lumbar spine, and others)
    \item \texttt{ImagingModality} — acquisition modalities (CT, MRI, PET, DXA, ultrasound, X-ray)
    \item \texttt{ImageRef} — representative verified figure references with provenance from the Vision Track
\end{itemize}

The direct-evidence edge policy is:

\begin{itemize}
    \item \texttt{(Microbe)-[:CORRELATES\_WITH]->(RadiomicFeature)}
    \item \texttt{(Microbe)-[:CORRELATES\_WITH]->(BodyCompositionFeature)}
    \item \texttt{(MicrobialSignature)-[:CORRELATES\_WITH]->(RadiomicFeature)}
    \item \texttt{(RadiomicFeature)-[:ASSOCIATED\_WITH]->(Disease)}
    \item \texttt{(BodyCompositionFeature)-[:ASSOCIATED\_WITH]->(Disease)}
    \item \texttt{(RadiomicFeature)-[:MEASURED\_AT]->(BodyLocation)}
    \item \texttt{(BodyCompositionFeature)-[:MEASURED\_AT]->(BodyLocation)}
    \item \texttt{(RadiomicFeature)-[:ACQUIRED\_VIA]->(ImagingModality)}
    \item \texttt{(BodyCompositionFeature)-[:ACQUIRED\_VIA]->(ImagingModality)}
    \item \texttt{(ImagingModality)-[:REPRESENTED\_BY]->(ImageRef)}
    \item \texttt{(Microbe)-[:CORRELATES\_WITH\_DISEASE]->(Disease)}
\end{itemize}

The relation type \texttt{ASSOCIATED\_WITH} is used for text-derived phenotype-to-disease edges to accurately reflect the associative, non-causal evidence level of text co-mention extraction. The relation type \texttt{CORRELATES\_WITH} is reserved for quantitatively verified subject-to-feature edges, specifically from the Vision Track where a correlation coefficient has been deterministically confirmed against the figure's color scale.

\subsection{Data Sourcing and Corpus}

Data was sourced from PubMed Central Open Access using a split query strategy with three lanes: a strict radiomics lane targeting papers that explicitly co-mention radiomics with microbiome markers; an adjacent imaging phenotype lane targeting CT and MRI body composition papers; and a clinical recall expansion lane. The final merged corpus totals \textbf{640 papers}, with PMC full text retrieved for open-access articles.

\subsection{Text Track: NER and Relation Extraction}

The Text Track operates as a multi-stage pipeline:

\begin{enumerate}
    \item \textbf{Phenotype extraction} (\texttt{src/extract\_radiomics\_text.py}): A vocabulary-matched NER module maps diverse terminology to canonical imaging phenotype names. Disease co-mentions are extracted with a two-layer filter that blocks section-header noise (``Abstract Background:''), population-context fragments (``patients with obesity''), and sentence-fragment disease strings.
    \item \textbf{Microbial and disease NER} (\texttt{src/text\_ner\_minerva.py}): The \texttt{d4data/biomedical-ner-all} model runs on Apple Silicon MPS for microbial entity extraction. Disease NER uses \texttt{en\_ner\_bc5cdr\_md}. Long sentences are chunked and batched.
    \item \textbf{Span cleanup} (\texttt{src/span\_cleanup.py}): A shared helper rejects wordpiece fragments (\texttt{\#\#} artifacts), generic microbial placeholder terms, trailing coordination fragments, and clause-prefix disease strings before relation extraction.
    \item \textbf{Relation extraction} (\texttt{src/relation\_extract\_stage.py}): Relations are extracted using Gemini 2.5 Flash-Lite via an OpenAI-compatible API, following MINERVA's self-consistency protocol: 7 samples per pair with full-consistency acceptance. The relation stage supports three backends (\texttt{heuristic}, \texttt{hf\_textgen}, \texttt{openai\_compatible}) and is intended to be run model-backed for production outputs.
    \item \textbf{Edge assembly} (\texttt{src/assemble\_edges.py}): Assembles graph-ready edges, enforces direct-evidence policy, emits audit-only bridge hypotheses, and exports Neo4j-ready CSV artifacts.
\end{enumerate}

\subsection{Vision Track: Figure-Aware Quantitative Extraction}

The Vision Track addresses the structured figure data gap. Pipeline stages are:

\begin{enumerate}
    \item \textbf{Figure indexing} (\texttt{src/index\_figures.py}): Indexes figures from PMC articles using the NCBI figure API.
    \item \textbf{VLM proposal} (\texttt{src/propose\_vision\_qwen.py}): Gemini 2.5 Flash-Lite proposes r-values for microbial entity to radiomic feature correlations from candidate heatmap figures.
    \item \textbf{Deterministic verification} (\texttt{src/verify\_heatmap.py}): Each VLM proposal is cross-checked against the figure's gradient color scale using pixel-level analysis. A proposal is accepted only when the predicted value's color is geometrically consistent with the scale legend and the support fraction of matching pixels meets the acceptance threshold.
\end{enumerate}

Across 3 figures tested: 1 was verified (PMC10605408, \textit{Prevotella nigrescens} $\leftrightarrow$ GLCM\_Correlation, CT, $r = 0.95$, distance\_metric$= 0.05$, support\_fraction$= 1.0$); 2 were correctly rejected (PMC10176953: dot/bar-plot style, not a gradient heatmap; PMC11924647: feature-to-feature matrix with no microbial entities).

On the full 640-paper corpus:

    \item \textbf{9} Gemini-validated \texttt{CORRELATES\_WITH\_DISEASE} microbe-disease pairs
    \item \textbf{18} \texttt{BodyLocation} nodes, \textbf{5} \texttt{ImagingModality} nodes

    \item \textbf{156} automated test checks passing
\end{itemize}

\subsection{Scope Decision: Body Composition Features}

The corpus evidence strongly supported extending scope from strict radiomic features to body composition features. MINERVA itself treats diseases such as sarcopenia and obesity as first-class nodes; the body composition literature demonstrates the same microbiome-phenotype coupling that radiomics does. The system therefore treats \texttt{RadiomicFeature} (texture, shape, intensity) and \texttt{BodyCompositionFeature} (sarcopenia, visceral adipose tissue, skeletal muscle index) as parallel nodes with identical edge types, rather than collapsing them into a single generic category.

\subsection{Known Gaps Relative to MINERVA Upstream}

\begin{itemize}
    \item \textbf{UMLS normalization}: MINERVA assigns a CUI to every entity via ScispaCy; the current system does not implement UMLS entity linking, so alias deduplication is not applied.
    \item \textbf{NER models}: The current system uses substitute models (\texttt{d4data/biomedical-ner-all} and \texttt{en\_ner\_bc5cdr\_md}) rather than MINERVA's custom BNER2.0-finetuned DistilBERT (F1 = 0.914). Professor-mediated access to BNER2.0 weights would allow a direct comparison run.
    \item \textbf{Relation model}: Gemini 2.5 Flash-Lite (zero-shot) is used rather than the BioMistral-AUG-7B model that MINERVA fine-tuned on 1,100 labeled sentences. Gemini's full-consistency rate on the current corpus is 0.896.
\end{itemize}
